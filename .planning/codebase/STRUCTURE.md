# Codebase Structure

**Analysis Date:** 2026-03-13

## Directory Layout

```text
company/
|-- .codex/                # GSD workflows, templates, and skills
|-- addons/                # Godot editor/runtime plugins
|-- assets/                # Raw image assets
|-- data/                  # Static game data resources (.tres)
|-- docs/                  # Design and migration documents
|-- scenes/                # Scene files and scene scripts
|-- src/                   # Core runtime/domain scripts
|-- project.godot          # Godot project config and autoload declarations
|-- AGENTS.md              # Collaboration constraints for this repo
`-- .editorconfig          # Minimal editor charset config
```

## Directory Purposes

**src/**
- Purpose: Primary game logic and framework modules.
- Contains: Domain folders (`save`, `inventory`, `expedition_system`, `player_progress`, `ui_framework`, etc.).
- Key files:
  - `src/save/SaveManager.gd`
  - `src/inventory/InventoryComponent.gd`
  - `src/expedition_system/expedition/runtime/ExpeditionRuntime.gd`
  - `src/player_progress/PlayerProgressRoot.gd`
  - `src/ui_framework/UIManager.gd`
- Subdirectories: Feature/domain oriented, each with related scripts/resources.

**scenes/**
- Purpose: Runtime scenes and manual test scene sets.
- Contains: `expedition/`, `invtest/`, `devtest/`, `attribute/`.
- Key files:
  - `scenes/expedition/ExpeditionSceneV2.tscn` (main runtime scene)
  - `scenes/invtest/inventorytest.tscn` (inventory test)
  - `scenes/devtest/TestHub.tscn` (generic dev test shell)

**data/**
- Purpose: Static content resources used by runtime/devtest.
- Contains: `data/devtest/...` actor, item, location, and inventory resources.
- Key files:
  - `data/devtest/expedition_v2/actors/default_actor_catalog.tres`
  - `data/devtest/expedition_v2/locations/forest_outpost_v2.tres`

**addons/**
- Purpose: Third-party/custom Godot plugins and native integration files.
- Contains:
  - `addons/imgui-godot/` (plugin, C# bridge, native binaries)
  - `addons/equipment-resource-creator/` (editor dock plugin)

## Key File Locations

**Entry Points:**
- `project.godot` - main scene and autoload registration.
- `scenes/expedition/ExpeditionSceneV2.tscn` - runtime start scene.

**Configuration:**
- `project.godot` - engine/project behavior.
- `.editorconfig` - encoding baseline (`utf-8`).
- `.gitignore` - excludes `.godot/` and `/android/`.

**Core Logic:**
- `src/save/` - save/load pipeline and codecs.
- `src/inventory/` - item container/session/component framework.
- `src/expedition_system/` - expedition and squad runtime.
- `src/player_progress/` - roster bootstrap and progress persistence.
- `src/ui_framework/` - global UI registry and panel lifecycle.

**Testing (manual):**
- `scenes/invtest/` - inventory integration tests.
- `scenes/devtest/` + `src/devtest/` - reusable panel-based test hub.
- `src/expedition_system/expedition/ui_debug/` - runtime debug controls.

**Documentation:**
- `AGENTS.md` - repository collaboration constraints.
- `src/*/README.md` - module-level design and usage notes.
- `docs/` - migration/planning docs.

## Naming Conventions

**Files:**
- GDScript class files typically use `PascalCase.gd` (for `class_name` scripts).
- Scene files typically use `PascalCase.tscn` for reusable scene types.
- Module READMEs use `README.md`.

**Directories:**
- Domain folders are mostly `snake_case` under `src/`.
- Scene and data groupings are purpose-driven (`devtest`, `invtest`, `expedition`).

**Special Patterns:**
- Godot metadata files use `.uid` alongside script/resource files.
- Save IDs follow path-like strings (for example `progress/player/roster`).

## Where to Add New Code

**New gameplay/domain feature:**
- Primary code: `src/<domain>/`
- Scene integration: `scenes/<domain>/`
- Data resources: `data/<environment>/<domain>/`

**Inventory-related behavior:**
- Core logic: `src/inventory/`
- Save codec changes: `src/save/codecs/`
- Manual regression scene: `scenes/invtest/`

**New UI panel:**
- Panel script/scene in `scenes/` (or matching domain folder).
- Register through `src/ui_framework/UIConfigTable.gd`.
- Open via `UIManager.show_ui(...)` rather than direct instantiate in business scripts.

**New dev/debug tool:**
- Scripts: `src/devtest/`
- Scenes/panels: `scenes/devtest/`
- Keep production behavior isolated from test-only hooks.

## Special Directories

**.godot/**
- Purpose: Godot generated cache/import metadata.
- Source: Generated by editor/runtime.
- Committed: No (ignored in `.gitignore`).

**addons/imgui-godot/bin/**
- Purpose: Native binaries for platform-specific plugin runtime.
- Source: Plugin distribution artifacts.
- Committed: Yes.
